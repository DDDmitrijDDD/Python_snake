Для создания игры Змейка были использованы следующие технологии и моменты:
1. Собственно Pygame
2. sys для проверки на ошибки и game over
3. random для случайного размещения еды и бонусной еды
4. os для чтения файлов, определяющих структуру уровня

Вся игра состоит из 5 классов:
1. Класс игры Game, в котором описаны все стартовые ресурсы(размер холста, начальное положение змеи,
        условия смены уровня, условия выигрыша и проигрыша, отображения результата и т.д.)
2. Класс стен Wall, показывающий, где будут размещены стены. Порядок размещения стен определяется текстовыми файлами
        в папке levels, размеры файлов 50х50 знаков. "#" - означает стену, "." - означает пустое место, "BBS" - змея.
3. Класс змеи Snake, описывающий начальное положение змеи, ее движение, прирост тела змеи при пересечении
        головы змеи с координатами еды. При этом пересечение с обычной едой проверяется сравнением, а пересечение с
        большой едой проверяется по пересечению масок объектов
4. Класс еды Food - инит еды и ее отображение
5. Класс большой(бонусной) еды Big_Food - инит и отображение бонусной еды. Кроме того этот класс определяет "таяние" еды -
        уменьшение еды в размерах и соответственно уменьшение очков, приносимых этой едой

Принцип игры прост - необходимо перемещаясь по экрану собирать еду(бордовые квадраты) и бонусную еду(синие круги)
с каждым собранным квадратом двигаться становится все сложнее, так как змея удлиняется

При наборе определенного количества очков, меняется уровень игры

Как можно улучшить проект:
1. Добавить возможность изменения стартового уровня вручную
2. Добавить возможность изменять количество уровней и рисовать уровни средствами Pygame
3. Добавить возможность хранения лучших результатов в отдельном файле
4. Добавить управляющие кнопки(хотя бы регулирующие скорость передвижения змеи)
5. Сделать фигуру змеи с помощью коллекций спрайтов, чтобы она была больше на змею похожа
   (хотя последнее в этой редакции проблематично)
